# Stat of the Day — Project Guide for Claude

## What This Is
A daily Premier League stat-grid puzzle game (clone of statpadgame.com), built in vanilla HTML/CSS/JS with no build step. Deployed via GitHub Pages at:
**https://aidanbdoyle.github.io/statpad-soccer/**

Players fill 5 rows by naming PL players that match each row's constraints (club, season range, qualifier). They try to maximise their total stat score, or in Target Mode hit a specific number.

## Script Load Order (important)
```html
<script src="puzzles.js"></script>
<script src="data/photo_codes.js"></script>
<script src="data/player_photos_extra.js"></script>
<script src="game.js"></script>
<!-- Inline: loads players_data.js with onload → init() -->
```
`init()` is called by the `onload` event on the dynamically-inserted `players_data.js` script tag — **not** a setTimeout. This was fixed after a race condition caused only 16 sample players to appear.

## Adding a New Puzzle

1. Open `puzzles.js`
2. Append a new object to the `PUZZLES` array (see `PUZZLE_TEMPLATE.md`)
3. Increment `puzzleNumber`
4. Set `date` to the corresponding calendar date (day 0 = 2026-04-15)
5. Commit and push — the game auto-selects by `daysSince % PUZZLES.length`

**The puzzle for a given day** = `PUZZLES[daysSince % PUZZLES.length]` where `daysSince` = days since 2026-04-15. If you run out of puzzles, it loops. Add puzzles to prevent looping.

## Puzzle Format Reference

### Top-level fields
| Field | Type | Notes |
|---|---|---|
| `puzzleNumber` | number | Sequential, shown in history and share text |
| `date` | string | `"YYYY-MM-DD"` — for reference only, not used by game logic |
| `category` | string | Display name: `"Goals"`, `"Appearances"`, `"Assists"`, `"Clean Sheets"` |
| `categoryKey` | string | DB key: `"goals"`, `"apps"`, `"assists"`, `"clean_sheets"` |
| `categoryUnit` | string | Singular unit: `"goal"`, `"appearance"`, `"assist"`, `"clean sheet"` |
| `categoryMode` | string | `"season"` (best single season) or `"career"` (sum of all valid seasons) |
| `target` | number | Target Mode number (set to a round achievable total) |

### Row fields
| Field | Type | Notes |
|---|---|---|
| `clubs` | array | Exact club name strings (see club list below). Empty `[]` = any club |
| `seasonStart` | number | Start year of the FIRST valid season (e.g. `2019` = 2019-20 season) |
| `seasonEnd` | number | Start year of the LAST valid season (e.g. `2026` = includes 2025-26) |
| `qualifier` | null / object / array | `null` = no qualifier. Array = player must satisfy ALL qualifiers (AND logic) |

## All Qualifier Types

### `nationality`
Single nationality match.
```js
{ type: "nationality", value: "France", display: "FRENCH", scopeDisplay: "CAREER" }
```

### `nationality_one_of`
OR logic across multiple nationalities (flags shown side by side).
```js
{ type: "nationality_one_of", values: ["Spain","Italy","Portugal"], display: "SPANISH / ITALIAN / PORTUGUESE", scopeDisplay: "SEASON" }
```

### `continent`
```js
{ type: "continent", value: "European", display: "EUROPEAN", scopeDisplay: "CAREER" }
// value options: "European" | "South American" | "African" | "Asian" | "North American"
```

### `position`
**Critical: use single-letter values from the player DB.**
```js
{ type: "position", value: "M", display: "MIDFIELDER", scopeDisplay: "SEASON" }
// value: "G" (goalkeeper), "D" (defender), "M" (midfielder), "F" (forward)
// ⚠️  NOT "GK", "DF", "MF", "FW" — the API stores single letters only
// GK is special-cased: value "GK" also works for goalkeepers specifically
```

### `outfield`
Excludes goalkeepers (no value needed).
```js
{ type: "outfield", display: "OUTFIELD PLAYER", scopeDisplay: "CAREER" }
```

### `award`
```js
// Golden Boot (top scorer that season)
{ type: "award", award: "golden_boot", scope: "season", display: "WON GOLDEN BOOT", scopeDisplay: "SAME SEASON" }
// scope "career" = won it at any point in their PL career
{ type: "award", award: "golden_boot", scope: "career", display: "WON GOLDEN BOOT", scopeDisplay: "CAREER" }

// PL Title
{ type: "award", award: "pl_title", scope: "season", display: "WON PL TITLE", scopeDisplay: "SAME SEASON" }
{ type: "award", award: "pl_title", scope: "career", display: "WON PL TITLE", scopeDisplay: "CAREER" }
```

### `relegated`
Player's team was relegated that season.
```js
{ type: "relegated", display: "RELEGATED", scopeDisplay: "SEASON" }
```

### `max_stat`
Caps how much of a stat the player can have (used to restrict to low-scorers, etc.).
```js
{ type: "max_stat", key: "goals", value: 5, scope: "career", display: "MAX 5 CAREER GOALS", scopeDisplay: "CAREER" }
// key: "goals" | "apps" | "assists" | "clean_sheets"
```

### `scopeDisplay` values
- `"SEASON"` — applies within the season range
- `"CAREER"` — applies across entire PL career
- `"SAME SEASON"` — award must be won in the same season they played for the club

## Player Data & Photos

### Player database (`data/players_data.js`)
- ~5,000+ players with full PL career history
- Generated by `scraper.py` (takes ~1-2 hours, scrapes fbref.com)
- Includes: name, nationality, position (single letter), seasons with clubs/stats

### Photo system (two-layer fallback)
1. **`data/photo_codes.js`** — FPL numeric codes for ~826 current-season players
   - URL format: `https://resources.premierleague.com/premierleague/photos/players/110x140/p{code}.png`
   - Pre-generated via `curl` from FPL bootstrap API (CORS blocks runtime fetch from GitHub Pages)
2. **`data/player_photos_extra.js`** — Wikipedia URLs for 60+ historical legends (Shearer, Henry, Cantona, etc.)
   - Format: `https://en.wikipedia.org/w/index.php?title=Special:Redirect/file/{filename}&width=110`
3. **Initials fallback** — shown if both above fail (graceful, never breaks)

To add more historical players to the photo map, edit `data/player_photos_extra.js`.

## Key Game Mechanics

### Scoring
- `categoryMode: "season"` → score = player's best single-season stat within the row's constraints
- `categoryMode: "career"` → score = sum of the stat across all valid seasons

### Percentile tiers (vs all valid answers for that row)
| Tier | Range | Colour |
|---|---|---|
| Purple | 100th (the single best answer) | `#7c3aed` |
| Blue | 95th–99th | `#2563eb` |
| Gold | 80th–94th | `#d97706` |
| Silver | 70th–79th | `#94a3b8` |
| Bronze | 60th–69th | `#b45309` |

### History & streak
- Stored in `localStorage` under key `statpad_history`
- Keyed by `YYYY-MM-DD` puzzle date (not today's date)
- First score per mode (normal/target) is locked; X button removes it
- Streak = consecutive calendar days with a completed puzzle, starting from 2026-04-15

### Puzzle date calculation
```js
daysSince = Math.floor((todayUTC - Date.UTC(2026, 3, 15)) / 86400000)
// Day 0 = 2026-04-15, Day 1 = 2026-04-16, etc.
```

## Known Gotchas

- **Position values are single letters** — `G`, `D`, `M`, `F` (not `GK`, `DF`, `MF`, `FW`). The player DB from the PL API stores them this way. GK is special-cased in `checkQualifier()`.
- **Accent-insensitive matching** — `normForAward()` strips accents for Golden Boot comparisons (fixes Agüero → Aguero mismatches). Player search also strips accents.
- **Club name exact match** — `"Nott'm Forest"` not `"Nottingham Forest"`. Copy from the list above.
- **seasonStart/seasonEnd are season start years** — `2019` means the 2019-20 season. `seasonEnd: 2026` includes 2025-26.
- **FPL photo CORS** — `fantasy.premierleague.com` blocks runtime fetches from GitHub Pages. `photo_codes.js` must be pre-generated via command line.
- **`nationality_one_of` = OR logic** — a plain array qualifier = AND logic. Don't confuse them.
- **`relegated` qualifier** — only works for `categoryMode: "season"` rows, since relegation is per-season.
- **Target number** — should be roughly achievable by a good player combination but hard to hit exactly. For Goals/season puzzles, 60-100 works well. For Appearances/career, 700-1000.
